# 05. 쿠폰 & 리퍼럴 시스템

## 목적
리퍼럴을 통해 사용자를 확보하고, 쿠폰으로 리텐션을 유지한다.

## 리퍼럴 시스템

### 추천 흐름
1. 사용자 A가 앱 설치 → 고유 추천 코드 자동 생성
2. A가 지인 B, C에게 추천 코드 공유
3. B, C가 앱 설치 후 A의 추천 코드 입력
4. A의 추천 카운트가 2가 되면 쿠폰 자격 획득

### 추천 코드
- 회원가입 시 UUID 기반 자동 생성
- 마이페이지에서 확인 및 복사 가능
- 공유 기능 (카카오톡, 문자, 링크 복사)

## 쿠폰 시스템

### 지급 조건
- 2명 추천 완료 필수
- 앱 설치 후 최소 7일 경과

### 지급 규칙
- 2주마다 3,000원 디지털 쿠폰 자동 지급
- 쿠폰 유효기간: 발행일로부터 30일

### 쿠폰 용도
- 유료 구독 할인 (Basic/Standard/Premium)
- 프리미엄 콘텐츠 개별 잠금해제

### 쿠폰 상태
- `active`: 사용 가능
- `used`: 사용 완료
- `expired`: 기간 만료 (자동 처리)

## 기술 구현

### Supabase Edge Function
- `process-referral`: 추천 코드 검증 및 처리

### 자동 쿠폰 지급
- Supabase Cron Job 또는 Edge Function 스케줄러
- 14일마다 조건 충족 사용자에게 쿠폰 자동 INSERT

### 만료 처리
- 일일 배치: expires_at < now() 인 active 쿠폰 → expired 변경

### 푸시 알림
- 쿠폰 지급 시 알림
- 쿠폰 만료 3일 전 알림
