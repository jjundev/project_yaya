# Mock 데이터 1차 표시 스펙

## 개요
Supabase 백엔드가 아직 준비되지 않은 상태에서도 사용자가 일일 운세 화면을 정상적으로 볼 수 있어야 한다. 현재는 사용자 프로필(생년월일, 성별)이 서버에서 불러와지지 않아 운세 화면에 빈 화면이 표시된다. 이 스펙은 백엔드 연동 여부와 무관하게 mock 데이터로 운세 화면이 항상 동작하도록 한다. Supabase가 준비되면 실제 데이터가 자연스럽게 우선 사용되어야 한다.

## 범위

### 포함
- 사용자 프로필 데이터(생년월일·성별)가 없을 때 mock 기본값으로 운세 로드
- 일일 운세 화면 정상 표시 (빈 화면 해소)
- 주간 운세 화면 정상 표시
- 사주 분석 카드 정상 표시

### 미포함 (다음 스프린트)
- 실제 Supabase 사용자 프로필 저장/조회
- Edge Function 배포 후 실제 AI 운세 생성
- 운세 캐싱 (DB fortunes 테이블 연동)

## 사용자 스토리

### 운세 화면 진입
- 사용자는 서버 연동 없이도 운세 탭에 진입했을 때 로딩 화면을 볼 수 있다
- 사용자는 로딩이 끝나면 오늘의 운세 카드(사랑·재물·건강·직업 점수 포함)를 볼 수 있다
- 사용자는 운세 카드를 탭했을 때 상세 운세 내용을 볼 수 있다

### 사주·오행 분석
- 사용자는 운세 홈 화면에서 오행 분석 카드를 볼 수 있다
- 사용자는 주간 운세 카드를 볼 수 있다

### 데이터 우선순위
- 사용자는 Supabase에 본인 프로필이 있을 경우 실제 데이터 기반 운세를 본다
- 사용자는 Supabase에 데이터가 없을 경우 mock 데이터 기반 운세를 본다 (빈 화면 표시 금지)

## 완료 조건 (Definition of Done)
- [ ] 신규 설치 직후(Supabase 미연동) 운세 탭 진입 시 빈 화면이 아닌 mock 운세가 표시된다
- [ ] 운세 카드 탭 → FortuneDetailView에서 운세 내용이 표시된다
- [ ] 사주 분석 카드가 홈 화면에 표시된다
- [ ] 주간 운세 카드가 홈 화면에 표시된다
- [ ] Supabase 연동 후(birthDate·gender non-nil)에도 기존 로직이 동일하게 동작한다

## 참고 사항
- `AIService`의 모든 AI 호출은 이미 mock 데이터를 반환하도록 구현돼 있음 (Edge Function 미배포 상태)
- Supabase DB 호출 실패(테이블 미존재 등)는 이미 `try?`로 조용히 처리 중
- 차단 지점은 `FortuneHomeView.loadData()`의 `birthDate`·`gender` nil 체크 하나뿐
