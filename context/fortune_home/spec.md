# 메인 홈 화면 스펙

## 개요

사주 분석 완료 후 사용자가 매일 열어보는 메인 화면이다. 핵심 목표는 **매일 재방문**이다. 사주 기반으로 AI가 생성한 오늘의 콘텐츠가 매일 달라지기 때문에, 어제와 다른 인사이트를 확인하러 다시 열게 만드는 것이 이 화면의 존재 이유다. 투자 기능은 별도 탭에서 다룬다. 잠긴 프리미엄 콘텐츠는 완전히 숨기지 않고 블러 미리보기로 노출해 구독 전환을 자연스럽게 유도한다.

## 범위

### 포함
- 개인화 헤더 (날짜 + 오늘의 에너지 한 줄)
- 오늘의 운세 카드 (무료, 매일 갱신)
- 오행 에너지 오늘의 상태 카드 (무료, 매일 갱신)
- 주간 운세 블러 미리보기 (Basic 이상 구독 유도)
- AI 사주 상담 진입 CTA (Premium 구독 유도)
- Pull-to-refresh 및 자정 자동 갱신

### 미포함 (다음 스프린트)
- 투자 에너지/운세 (별도 투자 탭에서 구현)
- 월간/연간 운세 미리보기 (주간 운세 출시 후 순차 추가)
- 소셜 공유 기능 (오늘의 운세 공유)
- 오늘의 운세 히스토리 조회

## 사용자 스토리

### 매일 재방문

- 사용자는 앱을 열었을 때 오늘 날짜와 자신의 사주 기반 에너지 한 줄 요약을 즉시 볼 수 있다
- 사용자는 매일 달라지는 운세 요약을 읽고 오늘 하루의 방향을 잡을 수 있다
- 사용자는 Pull-to-refresh로 운세를 즉시 갱신할 수 있다
- 사용자는 자정이 지나면 새 운세가 자동으로 로드된 것을 확인할 수 있다

### 오늘의 운세 확인 (무료)

- 사용자는 사랑/재물/건강/직장 4개 영역의 오늘 점수(1~5)를 시각적으로 볼 수 있다
- 사용자는 오늘 운세의 전체 흐름을 요약한 텍스트를 읽을 수 있다
- 사용자는 오늘의 조언 한 문장을 강조된 형태로 볼 수 있다
- 사용자는 오늘의 행운의 숫자와 행운의 색을 확인할 수 있다

### 오행 에너지 인사이트 (무료)

- 사용자는 자신의 오행 구성과 오늘 기운의 관계를 한 줄 인사이트로 볼 수 있다 (예: "오늘은 수(水) 기운이 강해 직관이 뛰어난 날이에요")
- 사용자는 오늘 자신의 에너지가 어떤 오행에 치우쳐 있는지 시각적으로 확인할 수 있다

### 프리미엄 전환 유도

- 사용자는 주간 운세의 첫 줄을 블러 처리된 상태로 볼 수 있다
- 사용자는 블러 미리보기를 탭하면 구독 페이지로 이동할 수 있다
- 사용자는 AI 사주 상담 진입 배너를 통해 Premium 구독 페이지로 이동할 수 있다
- 이미 구독 중인 사용자는 블러 없이 주간 운세 전체 내용을 바로 볼 수 있다

## 완료 조건 (Definition of Done)

- [ ] 앱 진입 시 오늘 날짜 + 개인화 에너지 한 줄이 헤더에 표시된다
- [ ] 오늘의 운세 카드가 4개 영역 점수, 요약, 조언, 행운 정보를 모두 포함한다
- [ ] 오행 에너지 인사이트 카드가 매일 다른 내용으로 표시된다
- [ ] 주간 운세 카드가 Free 사용자에게는 블러 미리보기로, Basic+ 사용자에게는 전체 내용으로 표시된다
- [ ] AI 상담 CTA가 Premium 구독 페이지로 연결된다
- [ ] Pull-to-refresh가 동작하고 새 데이터 로드 중 로딩 상태가 표시된다
- [ ] 자정 이후 첫 진입 시 전날 운세가 아닌 오늘 운세가 로드된다
- [ ] 운세 로드 실패 시 재시도 버튼이 표시된다

## 참고 사항

- **기존 화면**: `FortuneHomeView.swift`가 이 스펙의 구현 대상. 현재 오늘의 운세 카드, 사주 요약 카드, 잠금 카드(icon only)가 있음
- **개선 방향**: 잠금 카드 → 블러 미리보기로 교체 / 사주 요약 카드 → 오늘의 에너지 인사이트로 고도화
- **투자 기능 분리**: `InvestmentProfileView.swift`는 별도 탭에 위치하며 이 스펙에서 다루지 않음
- **AI 생성 콘텐츠**: 개인화 헤더의 에너지 한 줄, 오행 인사이트 문구는 Supabase Edge Function `daily-fortune`에서 생성
- **데이터 모델 참고**: `Fortune.swift`의 `FortuneContent`, `SajuAnalysis`, `FiveElements` 활용
