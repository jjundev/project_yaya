## 세부 체크 항목

### [필수] Product Depth — 스펙 기능이 실제로 동작하는가

- [ ] 히어로 섹션에 투자 타입 이모지, 타입명, 한 줄 설명이 모두 표시된다
  — FAIL 조건: 이모지 누락 / 타입명 하드코딩 / 설명 텍스트 빈 문자열

- [ ] `sajuBasis` 텍스트가 인용구 형식으로 표시된다
  — FAIL 조건: 일반 텍스트로 렌더링됨 / 섹션 자체가 없음 / 빈 문자열일 때 섹션이 표시됨

- [ ] 추천 ETF가 3개 이상 5개 이하로 목록에 표시된다
  — FAIL 조건: ETF 항목이 0개 / 6개 이상 표시 / ETF 이름 외 상세 설명이 함께 표시됨

- [ ] `strengths` 배열에서 최대 3개의 강점이 표시된다
  — FAIL 조건: 강점이 0개 표시 / 4개 이상 표시 / 항목이 고정 문자열로 하드코딩됨

- [ ] `risks` 배열에서 최대 2개의 리스크가 표시된다
  — FAIL 조건: 리스크가 0개 표시 / 3개 이상 표시

- [ ] 하단 CTA 버튼 텍스트가 "투자 시작하기"이다
  — FAIL 조건: "시작하기" 또는 다른 문구로 표시됨

### [필수] Functionality — 핵심 플로우가 막힘 없이 동작하는가

- [ ] `FirstFortuneResultView`의 "시작하기" 버튼을 탭하면 이 화면(`InvestmentOnboardingView`)으로 이동한다
  — FAIL 조건: `MainTabView`로 바로 이동 / 화면 전환 없음 / 앱 크래시

- [ ] "투자 시작하기" 버튼을 탭하면 `MainTabView`로 이동한다
  — FAIL 조건: 화면 전환 없음 / 이전 화면으로 돌아감 / 앱 크래시

- [ ] "투자 시작하기" 탭 시 `AuthViewModel.finishOnboarding()`이 호출된다
  — FAIL 조건: `needsOnboarding` 플래그가 false로 변경되지 않아 재진입 시 온보딩이 다시 시작됨

- [ ] `InvestmentProfile`이 nil일 때 앱이 크래시 없이 빈 상태 UI를 표시한다
  — FAIL 조건: 강제 언래핑으로 인한 크래시 / 화면 자체가 블랙스크린

- [ ] 이미 온보딩을 완료한 계정으로 로그인 시 이 화면이 표시되지 않는다
  — FAIL 조건: `MainTabView` 대신 `InvestmentOnboardingView`가 노출됨

- [ ] 4가지 투자 타입(공격형 / 안정형 / 가치투자형 / 성장추구형) 각각에서 화면이 크래시 없이 렌더링된다
  — FAIL 조건: 특정 타입에서 앱 크래시 / 빈 화면

### [권장] Visual Design — 인터페이스가 일관된 느낌인가

- [ ] 타입별 배경 색상이 서로 명확히 구분된다 (공격형🔥붉은/안정형🛡️파란/가치형💎금/성장형🚀보라)
  — FAIL 조건: 모든 타입이 동일한 단색(보라) 배경 / 색상 변화가 육안으로 구분 불가

- [ ] 섹션이 순차적으로 등장하는 애니메이션이 자연스럽게 느껴진다
  — FAIL 조건: 모든 섹션이 동시에 나타남 / 애니메이션 없이 정적으로 표시됨

- [ ] 히어로 이모지가 화면 상단에 충분히 크게 표시된다 (시각적 앵커 역할)
  — FAIL 조건: 이모지 크기가 다른 텍스트와 동일하거나 더 작음

- [ ] 하단 CTA 버튼이 스크롤과 무관하게 항상 하단에 고정되어 있다
  — FAIL 조건: 스크롤을 내려야 버튼이 보임 / 콘텐츠 최하단에 버튼이 붙어 있음

- [ ] 다크 모드에서 텍스트와 배경 간 가독성이 유지된다
  — FAIL 조건: 텍스트가 배경색과 겹쳐 읽기 불가

- [ ] `sajuBasis` 인용구가 시각적으로 일반 본문과 구분된다 (들여쓰기, 색상, 폰트 등)
  — FAIL 조건: 인용구가 일반 텍스트와 동일한 스타일로 렌더링됨

### [권장] Code Quality — 연결이 끊어진 곳은 없는가

- [ ] 화면에 표시되는 모든 텍스트가 `InvestmentProfile` 모델 데이터를 사용한다 (하드코딩 없음)
  — FAIL 조건: 임의 샘플 텍스트가 실제 사용자 데이터 대신 표시됨

- [ ] `strengths` 또는 `risks` 배열 항목이 3개 미만일 때 화면이 정상 렌더링된다
  — FAIL 조건: index out of range 크래시 / 빈 항목이 빈 셀로 표시됨

- [ ] ETF 목록이 비어 있을 때 ETF 섹션이 빈 리스트 대신 적절한 빈 상태로 처리된다
  — FAIL 조건: 빈 셀이 나열됨 / 섹션 타이틀만 표시되고 내용이 없음

- [ ] 이 화면이 `OnboardingFlowView`의 기존 플로우 밖에서 단독으로 진입되지 않는다
  — FAIL 조건: 딥링크 또는 다른 경로로 이 화면에 직접 도달 가능

### [선택] 추가 검증

- [ ] 화면 첫 등장 시 애니메이션 시작 전 빈 화면이 1초 이상 보이지 않는다
- [ ] 히어로 섹션과 첫 번째 스크롤 콘텐츠가 접힘 없이 첫 화면 내 노출된다 (iPhone SE 기준)
- [ ] VoiceOver 활성화 시 모든 섹션이 순서대로 읽힌다
