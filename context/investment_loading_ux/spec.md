# 투자 성향 로딩 UX 개선 스펙

## 개요

온보딩 마지막 단계에서 `InvestmentOnboardingView`가 "투자 성향을 불러오는 중이에요" 상태에서 진행되지 않는 문제가 있다.
현재 뷰는 `investmentProfile`이 nil인 경우 한 가지 빈 상태만 표시하며, 로딩 중인지 / 실패한 것인지 구분하지 않는다.
사용자는 오류가 발생했어도 계속 기다리다 이탈하게 된다.
이 스펙은 로딩·오류·성공 3가지 상태를 명확히 구분하여 사용자가 어떤 상황에서도 메인 화면으로 진입할 수 있도록 보장한다.

## 범위

### 포함
- 투자 성향 분석 **로딩 중** 상태 UI (진행 표시)
- 투자 성향 분석 **실패** 상태 UI (오류 안내 + 진행 허용)
- 어떤 상태에서도 "투자 시작하기" 버튼이 노출되어 사용자가 메인 화면으로 진행 가능
- `InvestmentOnboardingView`에 로딩·오류 상태 정보 전달

### 미포함 (다음 스프린트)
- 오류 발생 시 재시도(retry) 버튼
- 투자 성향 분석 백그라운드 재시도 로직
- 메인 화면에서 투자 성향이 없을 때의 fallback UX

## 사용자 스토리

### 로딩 상태
- 사용자는 투자 성향 분석이 진행 중일 때 로딩 중임을 알리는 시각적 표시를 볼 수 있다
- 사용자는 분석이 완료되기를 기다리는 중에도 "투자 시작하기" 버튼을 눌러 메인 화면으로 진입할 수 있다

### 오류 상태
- 사용자는 투자 성향 분석이 실패했을 때 "아직 분석을 불러오지 못했어요" 같은 안내 메시지를 볼 수 있다
- 사용자는 분석 실패와 무관하게 "투자 시작하기" 버튼을 눌러 메인 화면으로 진입할 수 있다
- 사용자는 메인 화면에서 투자 성향을 나중에 확인할 수 있다는 안내를 볼 수 있다

### 성공 상태 (기존 동작 유지)
- 사용자는 투자 성향이 로드되면 기존 프로필 콘텐츠(히어로, ETF, 강점 등)를 순차 애니메이션으로 볼 수 있다

## 완료 조건 (Definition of Done)

- [ ] 로딩 중(`isLoading = true`)일 때 로딩 인디케이터가 표시된다
- [ ] 로딩 실패(`errorMessage != nil`)일 때 "아직 투자 성향을 불러오지 못했어요" 안내 메시지가 표시된다
- [ ] 로딩 중 / 실패 / 성공 세 가지 상태 모두에서 "투자 시작하기" 버튼이 노출된다
- [ ] "투자 시작하기" 버튼 탭 시 메인 화면으로 정상 진입한다
- [ ] `InvestmentOnboardingView`가 로딩·오류 상태를 외부로부터 전달받는다
- [ ] 기존 성공 시 순차 애니메이션 동작이 그대로 유지된다

## 참고 사항

### 현재 상태 흐름
- `InvestmentViewModel`은 `isLoading: Bool`과 `errorMessage: String?`을 이미 관리한다
- `InvestmentOnboardingView`는 현재 `investmentProfile: InvestmentProfile?`만 전달받는다
- `emptyView`의 `onAppear`에서 `showButton = true`를 설정하고 있으나, 로딩/오류 구분이 없다

### UX 원칙
- 사용자가 오류 때문에 막히는 일이 없도록 "진행 가능성"을 항상 보장한다
- 오류 메시지는 기술적 내용을 노출하지 않고 사용자 친화적으로 표현한다
- 이 화면의 감정적 피크 경험(성공 케이스)은 기존 디자인 그대로 유지한다
