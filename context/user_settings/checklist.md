## 평가 기준 요약

| 기준 | 가중치 | FAIL 임계값 |
|------|--------|------------|
| Product Depth | 높음 | [필수] 항목 중 stub 또는 미구현 1건 이상 |

## 세부 체크 항목

### [필수] Product Depth — 스펙 기능이 실제로 동작하는가

#### 사주 재분석
- [ ] ProfileView 설정 섹션에 "사주 재분석" 항목이 표시된다 — FAIL 조건: 항목이 목록에 없거나 Text 플레이스홀더로 표시됨
- [ ] "사주 재분석" 탭 시 확인 다이얼로그가 나타난다 — FAIL 조건: 다이얼로그 없이 바로 이동하거나 아무 반응 없음
- [ ] 다이얼로그에서 "확인" 탭 시 온보딩 첫 화면(성별 선택)으로 이동한다 — FAIL 조건: 이동하지 않거나 다른 화면으로 이동함
- [ ] 다이얼로그에서 "취소" 탭 시 프로필 화면에 머문다 — FAIL 조건: 화면이 전환되거나 앱이 비정상 종료됨
- [ ] 재분석 확인 후 기존 사주/투자 데이터가 초기화된다 — FAIL 조건: 온보딩 완료 후 이전 사주 데이터가 그대로 남아 있음

#### 알림 설정
- [ ] 알림 설정 화면에 일일 운세 알림 on/off 토글이 표시된다 — FAIL 조건: 화면이 비어 있거나 Text 플레이스홀더로 표시됨
- [ ] 알림 토글을 처음 켤 때 iOS 시스템 알림 권한 요청이 표시된다 — FAIL 조건: 권한 요청 없이 토글만 켜짐
- [ ] 알림이 켜진 상태에서 수신 시간 피커가 활성화된다 — FAIL 조건: 피커가 표시되지 않거나 비활성화 상태로 고정됨
- [ ] 알림이 꺼진 상태에서 수신 시간 피커가 비활성화(또는 숨김)된다 — FAIL 조건: 알림 off 상태에서도 피커가 조작 가능함
- [ ] 알림 on/off 상태가 앱 재시작 후에도 유지된다 — FAIL 조건: 앱 종료 후 재시작 시 설정값이 초기화됨
- [ ] 수신 시간 설정값이 앱 재시작 후에도 유지된다 — FAIL 조건: 앱 종료 후 재시작 시 시간이 기본값(오전 8시)으로 초기화됨
- [ ] 시스템에서 알림을 거부한 경우 설정 앱으로 이동하는 안내가 표시된다 — FAIL 조건: 거부 상태임에도 안내 없이 토글만 켜짐

#### 이용약관 / 개인정보처리방침
- [ ] 이용약관 탭 시 Safari가 열린다 — FAIL 조건: 아무 반응 없거나 앱 내 빈 화면이 표시됨
- [ ] 개인정보처리방침 탭 시 Safari가 열린다 — FAIL 조건: 아무 반응 없거나 앱 내 빈 화면이 표시됨

#### 로그아웃
- [ ] 로그아웃 버튼 탭 시 확인 다이얼로그가 나타난다 — FAIL 조건: 다이얼로그 없이 즉시 로그아웃됨
- [ ] 다이얼로그에서 "확인" 탭 시 로그아웃 후 로그인 화면으로 이동한다 — FAIL 조건: 로그아웃되지 않거나 메인 화면에 그대로 남음
- [ ] 다이얼로그에서 "취소" 탭 시 현재 화면에 머문다 — FAIL 조건: 취소했는데 로그아웃되거나 화면이 전환됨

---

### [필수] Functionality — 핵심 플로우가 막힘 없이 동작하는가

- [ ] 사주 재분석 → 온보딩 완료 → 메인 화면 진입까지 전체 플로우가 막힘 없이 동작한다 — FAIL 조건: 어느 단계에서든 크래시 또는 UI 멈춤 발생
- [ ] 알림 설정 화면 진입 및 토글 조작 중 크래시가 발생하지 않는다 — FAIL 조건: 토글 조작 시 앱 비정상 종료
- [ ] 알림 권한 거부 후 재시도 시 앱이 정상 동작한다 — FAIL 조건: 권한 거부 후 앱이 멈추거나 토글 상태가 불일치함
- [ ] 로그아웃 후 재로그인하면 메인 화면으로 정상 진입한다 — FAIL 조건: 재로그인 후 화면이 비어있거나 크래시 발생

---

### [권장] Visual Design — 인터페이스가 일관된 느낌인가

- [ ] 사주 재분석 항목의 아이콘과 레이블이 기존 설정 항목과 시각적으로 일관된다 — FAIL 조건: 아이콘 없거나 글자 크기/색상이 다른 항목과 다름
- [ ] 알림 설정 화면의 레이아웃이 앱 전체 디자인 톤과 일치한다 — FAIL 조건: 시스템 기본 스타일이 그대로 노출되어 이질감이 있음
- [ ] 수신 시간 피커가 비활성화 상태일 때 시각적으로 구분된다 (흐림 처리 등) — FAIL 조건: 비활성화 상태와 활성화 상태가 동일하게 보임
- [ ] 확인 다이얼로그(사주 재분석, 로그아웃)의 버튼 레이블이 명확하다 ("확인"/"취소" 또는 동등한 문구) — FAIL 조건: 버튼 레이블이 없거나 의미가 불분명함

---

### [권장] Code Quality — 연결이 끊어진 곳은 없는가

- [ ] 빌드가 warning 없이 성공한다 — FAIL 조건: 빌드 실패 또는 새로 추가된 코드에서 warning 발생
- [ ] 알림 설정값이 `@AppStorage`로 저장되어 UserDefaults에 올바르게 유지된다 — FAIL 조건: `@State`로만 관리되어 앱 재시작 시 초기화됨
- [ ] 사주 재분석 시 `sajuAnalysis`, `investmentProfile` 양쪽이 모두 초기화된다 — FAIL 조건: 한쪽만 초기화되어 온보딩 완료 후 데이터 불일치 발생
- [ ] 이용약관/개인정보처리방침 URL이 `AppConfig.swift` 상수로 관리된다 — FAIL 조건: URL이 뷰 코드에 하드코딩됨
- [ ] 로그아웃 확인 다이얼로그가 기존 `AuthViewModel.signOut()` 함수를 재사용한다 — FAIL 조건: signOut 로직이 중복 구현됨

### [선택] 추가 검증

- [ ] 알림 시간 피커의 기본값이 오전 8시로 설정된다
- [ ] 이용약관 URL이 유효한 주소를 가리킨다 (빈 문자열이나 예시 URL이 아님)
- [ ] 사주 재분석 다이얼로그에 "기존 운세 기록이 초기화됩니다" 안내 문구가 포함된다
