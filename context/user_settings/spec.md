# 사용자 설정 스펙
> Planner 작성 | 참조: Generator, Evaluator

## 개요
앱 사용 중 사용자가 자신의 경험을 관리할 수 있는 설정 기능을 완성한다.
현재 ProfileView 설정 섹션에 네비게이션 링크만 있고 실제 뷰가 없는 항목(알림 설정, 이용약관, 개인정보처리방침)을 구현하고,
사주 재분석(온보딩 초기화)과 로그아웃 확인 다이얼로그를 추가한다.
이를 통해 사용자는 앱을 처음부터 다시 설정하거나, 알림 수신 여부와 시간을 직접 제어할 수 있게 된다.

## 범위
### 포함
- **사주 재분석**: 설정 섹션에 메뉴 추가 → 확인 다이얼로그 → 사주/투자 데이터 초기화 → 온보딩 첫 화면으로 이동
- **알림 설정**: 일일 운세 알림 on/off 토글 + 수신 시간 피커(기본: 오전 8시). 알림 on 시 iOS 시스템 권한 요청
- **이용약관**: 탭 시 Safari로 이용약관 URL 열기
- **개인정보처리방침**: 탭 시 Safari로 개인정보처리방침 URL 열기
- **로그아웃 확인 다이얼로그**: 기존 로그아웃 버튼 탭 시 확인 단계 추가

### 미포함 (다음 스프린트)
- 알림 카테고리별 분리 (주간/월간 운세 별도 알림)
- 이용약관/개인정보처리방침 앱 내 웹뷰
- 계정 탈퇴

## 사용자 스토리
### 사주 재분석
- 사용자는 프로필 탭 설정 섹션에서 "사주 재분석" 항목을 볼 수 있다
- 사용자는 "사주 재분석"을 탭하면 "다시 분석하면 기존 운세 기록이 초기화됩니다" 확인 다이얼로그를 볼 수 있다
- 사용자는 확인 후 온보딩 첫 화면(성별 선택)으로 이동할 수 있다
- 사용자는 취소를 선택하면 현재 프로필 화면에 머무를 수 있다

### 알림 설정
- 사용자는 알림 설정 화면에서 일일 운세 알림을 on/off할 수 있다
- 사용자는 알림을 처음 켤 때 iOS 시스템 알림 권한 요청을 받는다
- 사용자는 알림이 켜진 상태에서 수신 시간을 선택할 수 있다 (시/분 피커)
- 사용자는 시간을 변경하면 즉시 반영됨을 인지할 수 있다 (저장 버튼 불필요)
- 사용자는 알림 설정값이 앱을 껐다 켜도 유지됨을 경험할 수 있다
- 사용자는 시스템 설정에서 알림을 거부한 경우, 설정 앱으로 이동하는 안내를 볼 수 있다

### 이용약관 / 개인정보처리방침
- 사용자는 프로필 설정에서 이용약관을 탭하면 Safari에서 이용약관을 볼 수 있다
- 사용자는 프로필 설정에서 개인정보처리방침을 탭하면 Safari에서 내용을 볼 수 있다

### 로그아웃
- 사용자는 로그아웃 버튼을 탭하면 "정말 로그아웃할까요?" 확인 다이얼로그를 볼 수 있다
- 사용자는 확인 후 로그인 화면으로 이동할 수 있다
- 사용자는 취소를 선택하면 현재 화면에 머무를 수 있다

- [ ] 확인 시 기존 사주/투자 데이터가 초기화되고 온보딩 첫 화면으로 이동한다
- [ ] 알림 설정 화면에 on/off 토글이 있다
- [ ] 알림 on 시 iOS 시스템 권한 요청이 표시된다
- [ ] 알림이 켜진 상태에서 수신 시간 피커가 활성화된다
- [ ] 알림 on/off 상태와 시간이 앱 재시작 후에도 유지된다
- [ ] 시스템에서 알림 거부 시 설정 앱으로 이동하는 안내가 표시된다
- [ ] 이용약관 탭 시 Safari가 열린다
- [ ] 개인정보처리방침 탭 시 Safari가 열린다
- [ ] 로그아웃 버튼 탭 시 확인 다이얼로그가 표시된다
- [ ] 확인 시 로그아웃 후 로그인 화면으로 이동한다

## 참고 사항
- 로그아웃 함수: `AuthViewModel.signOut()` 이미 구현됨 (`AuthViewModel.swift:164`)
- 로그아웃 버튼: `ProfileView.swift:131`에 존재 — 확인 다이얼로그만 추가
- 온보딩 재진입: `AuthViewModel.needsOnboarding = true` 설정으로 온보딩 화면 노출 가능
- 알림 설정 저장: `@AppStorage` 사용 (알림 on/off `Bool`, 수신 시간 `Date`)
- 이용약관/개인정보처리방침 URL: `AppConfig.swift`에 상수로 추가할 것
- 설정 섹션 위치: `ProfileView.swift` — 기존 "알림 설정", "이용약관", "개인정보처리방침" NavigationLink 대상 뷰를 실제 구현으로 교체
