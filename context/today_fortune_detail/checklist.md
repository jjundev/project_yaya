# 오늘의 운세 상세 화면 체크리스트

## 평가 기준 요약

| 기준 | 가중치 | FAIL 임계값 |
|------|--------|------------|
| Product Depth | 높음 | [필수] 항목 중 stub 또는 미구현 1건 이상 |
| Functionality | 높음 | [필수] 항목 중 크래시 또는 플로우 막힘 1건 이상 |
| Visual Design | 중간 | [권장] Visual 항목 중 FAIL 3건 이상 |

## 세부 체크 항목

### [필수] Product Depth — 스펙 기능이 실제로 동작하는가

- [ ] FortuneHomeView의 오늘의 운세 카드를 탭하면 상세 화면으로 이동한다 — FAIL 조건: 탭해도 반응 없거나 다른 화면으로 이동
- [ ] 상세 화면 헤더에 오늘 날짜와 전체 운세 요약 텍스트가 표시된다 — FAIL 조건: 날짜 또는 요약 중 하나라도 누락
- [ ] 사랑운 섹션에 점수(1~5)와 해설 텍스트가 표시된다 — FAIL 조건: 점수 또는 해설 텍스트 누락
- [ ] 재물운 섹션에 점수(1~5)와 해설 텍스트가 표시된다 — FAIL 조건: 점수 또는 해설 텍스트 누락
- [ ] 건강운 섹션에 점수(1~5)와 해설 텍스트가 표시된다 — FAIL 조건: 점수 또는 해설 텍스트 누락
- [ ] 직장운 섹션에 점수(1~5)와 해설 텍스트가 표시된다 — FAIL 조건: 점수 또는 해설 텍스트 누락
- [ ] AI 개인화 메시지(편지) 섹션이 표시된다 — FAIL 조건: 섹션 자체가 없거나 빈 텍스트
- [ ] 행운의 숫자가 표시된다 — FAIL 조건: 숫자 미표시
- [ ] 행운의 색이 표시된다 — FAIL 조건: 색 미표시
- [ ] 공유 버튼을 탭하면 시스템 공유 시트(UIActivityViewController)가 호출된다 — FAIL 조건: 버튼 없음 또는 탭해도 공유 시트 미표시

### [필수] Functionality — 핵심 플로우가 막힘 없이 동작하는가

- [ ] 상세 화면에서 뒤로가기를 탭하면 FortuneHomeView로 돌아온다 — FAIL 조건: 뒤로가기 불가 또는 크래시
- [ ] 운세 데이터가 아직 로드되지 않은 상태에서 상세 화면 진입 시 로딩 표시 또는 빈 상태 처리가 된다 — FAIL 조건: 크래시 또는 빈 화면(안내 없음)
- [ ] dailyFortune이 nil인 상태에서 상세 화면 진입 시 크래시 없이 안내 메시지를 표시한다 — FAIL 조건: 크래시 또는 force unwrap 에러
- [ ] FortuneViewModel의 기존 dailyFortune 데이터를 재사용하며 별도 API 호출을 하지 않는다 — FAIL 조건: 상세 화면 진입 시 새로운 네트워크 요청 발생

### [권장] Visual Design — 인터페이스가 일관된 느낌인가

- [ ] 상세 화면의 색상 톤이 FortuneHomeView와 일관성 있다 (보라색 그라데이션 + 카드 스타일) — FAIL 조건: 전혀 다른 색상 체계 사용
- [ ] 영역별 점수 시각화가 직관적이다 (도트, 별, 바 등 시각 요소 사용) — FAIL 조건: 숫자만 텍스트로 표시하고 시각 요소 없음
- [ ] AI 개인화 메시지 섹션이 다른 섹션과 시각적으로 구분된다 (배경색, 테두리 등) — FAIL 조건: 다른 섹션과 전혀 구분 불가
- [ ] 스크롤이 자연스럽고 컨텐츠가 화면 밖으로 잘리지 않는다 — FAIL 조건: 컨텐츠 잘림 또는 스크롤 불가

### [권장] Code Quality — 연결이 끊어진 곳은 없는가

- [ ] 빌드가 warning 없이 성공한다 — FAIL 조건: 빌드 실패 또는 신규 warning 발생
- [ ] 상세 화면 View에 대한 XCTest(단위 테스트)가 존재한다 — FAIL 조건: 테스트 파일 없음
- [ ] FortuneContent 모델 변경 시 기존 테스트가 깨지지 않는다 — FAIL 조건: 기존 XCTest FAIL

### [선택] 추가 검증

- [ ] 다크 모드에서 상세 화면이 정상적으로 표시된다
- [ ] Dynamic Type(큰 글씨) 설정에서 레이아웃이 깨지지 않는다
- [ ] VoiceOver 접근성 라벨이 주요 요소에 설정되어 있다
